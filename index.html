记一次基于Spring Cloud的分布式事务解决方案Seata的整合过程

Posted on 2022-10-03 Edited on 2022-10-04 In Java

前言

何为Seata？一个阿里巴巴的一个开源的分布式事务解决方案项目，致力于解决分布式环境下的事务问题，官网为https://seata.io/，其基本原理如下图所示：

大致原理为所有的微服务都需要向TC（Transaction Coordinator）服务端注册，TC服务端会全局控制微服务事务调用关系的整个过程，包括事务进行间的数据管理；TM（Transaction Manager）端则负责全局事务的创建及管理；RM（Resource Manager）为各分支事务；每次调用事务发生，触发TM，就会产生一个全局事务id，为xid，这个全局事务id会在整个全局事务下的个分支事务间传递，以达到通知和控制管理效果，事务进行期间各分支事务都会把相应的sql log传到TC端，TC端负责管理起来，若期间任一事务发生异常，TC收到消息，便通知各分支事务进行从之前产生的sql log进行rollback补偿，以达到全局事务一致性目的；总的来说就是把分布式事务进行归一化，集中控制、管理、反馈。Seata主要把上述环境的如TC、TM环境搭建起来（要进行服务发现和中心化配置及数据存储管理等），然后在事务方法上加上@GlobalTransactional注解即可。本文将Seata与Nacos配合完成环境搭建和测试，Seata的Github发布地址为https://github.com/seata/seata/releases
